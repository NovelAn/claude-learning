## 常见问题

### Q1: 我应该使用 Chat Mode 还是直接描述任务？
**A**:
- **使用 Chat Mode**：当你需要逐步完善功能、探索不同方案、或者不确定具体实现时
- **直接描述**：当需求清晰、目标明确、希望快速得到完整方案时

### Q2: 如何避免上下文窗口超限？
**A**:
1. 优先引用必要的文件
2. 使用 `@directory` 查看结构而非内容
3. 分阶段请求，避免一次性请求太多
4. 使用 `/clear` 清除不需要的上下文

### Q3: Plan Mode 什么时候应该退出？
**A**:
- 计划已经足够详细
- 你对计划满意并准备开始实施
- 计划已经被验证可行
- 不需要进一步的架构讨论

### Q4: 如何提高 Claude Code 的回答质量？
**A**:
1. 提供清晰的上下文和背景
2. 明确说明约束和要求
3. 提供示例和参考代码
4. 使用专业术语但避免过度缩写
5. 分步骤描述复杂任务
